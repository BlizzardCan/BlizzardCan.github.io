## 从文本到向量：Embedding 的魔法

在 MindBridge 的 RAG 系统中，核心问题是：**怎样让计算机理解文本的含义，而不仅仅是字符匹配？**

答案是：**Embedding（嵌入向量）**。

### 什么是 Embedding？

简单说，Embedding 就是把文本转换成计算机能理解的 **稠密向量**。

```
用户问题：「最近总是焦虑睡不着怎么办？」
          ↓
      Embedding 模型
          ↓
向量表示：[0.23, -0.15, 0.89, 0.12, ..., -0.44]  （1536 维）
```

这个向量捕捉了原文本的 **语义含义**——"焦虑"、"失眠"、"寻求帮助"的信息都被编码进这个向量里。

### 为什么是 1536 维？

MindBridge 选择了 OpenAI 的 `text-embedding-3-small` 模型，生成 **1536 维向量**。

为什么不选更高维度（比如 10240 维）？

- ✅ **1536 维**：维度足够表达语义，存储成本低，检索速度快
- ❌ **10240 维**：虽然理论上更精准，但会导致向量库存储压力爆炸，检索速度变慢

对于心理咨询知识库的量级，1536 维完全够用，性价比最高。

### 向量空间中的相似性度量

现在有了向量，下一个问题：**怎样判断两个向量的相似度？**

常见的算法有 3 种：

#### 1. 欧氏距离（Euclidean Distance）

```
distance = √[(x₁-x₂)² + (y₁-y₂)² + ... + (z₁-z₂)²]
```

**含义**：向量在多维空间中的直线距离

**缺点**：
- 受向量长度（模长）影响很大
- 文本 Embedding 本身就是单位化的，长度不同反而是噪声
- 不适合文本语义匹配

#### 2. 点积相似度（Dot Product）

```
similarity = x₁·x₂ + y₁·y₂ + ... + z₁·z₂
```

**含义**：两个向量的方向和大小综合影响

**缺点**：
- 没做归一化，不够稳定
- 向量长度也会影响结果（不是我们想要的）

#### 3. 余弦相似度（Cosine Similarity）⭐

```
cos(θ) = (A · B) / (|A| × |B|)
```

**含义**：两个向量在多维空间中的**夹角余弦值**

**计算步骤**：
1. 计算两个向量的点积
2. 计算两个向量的模长
3. 点积 / （模长1 × 模长2）

**结果范围**：-1 到 1（对于 Embedding 通常是 0 到 1）

### 为什么余弦相似度最适合文本检索？

#### 关键优势 1：只关注方向，不关注长度

```
向量 A：[0.5, 0.5]  长度 = √(0.5²+0.5²) = 0.707
向量 B：[1.0, 1.0]  长度 = √(1.0²+1.0²) = 1.414

方向完全相同（都是 45° 角）

欧氏距离：√[(1.0-0.5)² + (1.0-0.5)²] = 0.707  ❌ 表示不同
余弦相似度：1.0  ✅ 表示完全相同
```

在文本 Embedding 中，**长度差异不代表语义差异**，我们只关心方向是否一致。

#### 关键优势 2：解释性强

- 余弦相似度 = 1：完全相同语义
- 余弦相似度 = 0.8：很相似
- 余弦相似度 = 0.5：中等相似
- 余弦相似度 = 0：完全无关

直观易懂，便于调试和优化。

#### 关键优势 3：计算效率高

Embedding 通常是**单位向量**（已经归一化），那么：

```
cos(θ) = A · B / (1 × 1) = A · B
```

计算就直接变成点积！极其高效。

这正是 **Chroma 向量库原生支持余弦相似度** 的原因——它基于海量向量检索的场景优化，追求极致性能。

### 实际对比

```
用户问题向量：「焦虑睡眠问题」→ v_user
知识库文档向量：「失眠症状及调适」→ v_doc

余弦相似度 = 0.87  ✅ 非常高的匹配度

如果用欧氏距离：
可能因为向量长度差异，给出低相似度  ❌
```

### 我的选择

在 MindBridge 中，我选择了 **余弦相似度**，原因总结：

1. **数学原理**：只关注方向，符合文本语义的本质
2. **性能最优**：结合 1536 维 Embedding，检索毫秒级响应
3. **生态支持**：Chroma 原生优化，不需要自己从零实现
4. **业务适配**：心理咨询知识库的量级和检索频率，完全不需要更复杂的算法

## 关键启示

**好的算法不是最复杂的，而是最契合场景的。**

在我评估其他算法（点积、欧氏距离）后，最终还是选了最成熟的余弦相似度。这背后的思路是：

- ✅ 数学原理正确
- ✅ 性能指标可控
- ✅ 生态成熟稳定
- ✅ 便于后续优化调试

这个选择让 MindBridge 的检索系统稳定运行，支撑了后续的重排、生成等高层逻辑。

**技术选型最重要的是 balance：最适合的，而不是最先进的。**
