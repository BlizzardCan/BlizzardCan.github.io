## Prompt：给 LLM 的 "指令 + 素材"

在 MindBridge 的 RAG 流程中，经历了粗排、精排、获得精准知识后，最后一步是 **拼接 Prompt 给大模型**。

这一步看似简单，其实是 **最容易出现幻觉的地方**。

一个设计不当的 Prompt，会导致模型忽视知识库，自行编造答案。

这篇文章主要整理我在 Prompt 上踩过的坑：信息太多会降质，约束太弱会幻觉，所以最后要走“少而精 + 强约束”的路线。

<!--more-->

### 什么是 Prompt？

Prompt 就是一段 **精心设计的指令 + 上下文 + 任务描述**，告诉大模型：

> "你是谁"、"你的职责是什么"、"这是你要处理的信息"、"请按照以下规则生成回答"

### 我在 MindBridge 中的 Prompt 优化

经过多轮迭代，我将 Prompt 优化到以下结构：

#### 优化点 1：加严格的系统约束

**优化前**（会幻觉）：
```
你是心理咨询助手，请根据用户问题提供建议。

用户问题：最近特别焦虑，该怎么办？
相关知识：[Chunk 1, Chunk 2, ...]

请给出建议。
```

问题：模型看到"心理咨询助手"，可能会用自己的知识库开始编造。

**优化后**（约束严格）：
```
你是校园心理服务助手。

你的职责：
1. 仅根据提供的知识库内容回答问题
2. 不知道的内容直接说「暂无相关心理援助信息」
3. 不编造、不推测、不补充知识库以外的建议
4. 回答简洁专业

用户问题：最近特别焦虑，该怎么办？

知识库相关内容：
[Chunk 1]：焦虑症的症状表现...
[Chunk 2]：焦虑症的缓解方法...
[Chunk 3]：何时需要寻求专业帮助...

请严格基于上述内容生成回答。如果内容不足以回答，直接说「暂无」。
```

**效果**：这样的约束让模型的行为完全可控，幻觉率大幅下降。

#### 优化点 2：控制知识库 Chunk 的数量

这一步我花了很长时间才理解。

**现象**：我给模型 Top20 的知识库文档，结果回答质量反而下降了。

**原因**：LLM 在处理过长的上下文时，会出现 **"遗忘中间信息"的问题**（特别是开源 7B 模型）。

给它太多 Chunk，反而会让模型迷茫：

```
模型的思考过程：
"这么多信息，哪个最重要？"
"好像都有点相关..."
"算了，我自己组织一下回答吧"（开始幻觉）
```

**优化方案**：精排后只取 Top5

```
Rerank 结果：
[Chunk A: 0.92]  ← Top1，明显相关
[Chunk B: 0.88]  ← Top2
[Chunk C: 0.85]  ← Top3
[Chunk D: 0.82]  ← Top4
[Chunk E: 0.79]  ← Top5
[Chunk F: 0.71]  ← 不要了
...

只把 Top5 送给模型，信息精而不冗。
```

**效果**：
- ✅ Prompt 长度可控（不超过模型最优上下文窗口）
- ✅ 模型易于集中注意力
- ✅ 生成质量明显提升
- ✅ Token 消耗减少（推理成本降低）

#### 优化点 3：精简上下文历史

多轮对话需要保留历史上下文（保证对话连贯性）。

但不能无限保留，否则 Prompt 会爆炸。

**我的策略**：只保留最近 10 轮对话

```
用户第 1 轮：「最近特别焦虑」
Assistant 第 1 轮：「可以尝试...」
用户第 2 轮：「还是睡不着」
Assistant 第 2 轮：「建议...」
...
用户第 10 轮：「现在感觉怎样？」（保留这些）
Assistant 第 10 轮：「...」

用户第 1-9 轮的对话：删除（节省空间）
```

**为什么是 10 轮？**

- ✅ 足够保留足够的对话语境
- ✅ 不会导致 Prompt 过长
- ✅ 对于短期咨询场景够用

### 完整 Prompt 的最终样子

经过 3 个优化后，我在 MindBridge 中的 Prompt 结构是这样的：

```
=== 系统指令 ===
你是校园心理服务助手。

职责：
- 仅根据提供的知识库回答问题
- 不知道就说「暂无相关心理援助信息」
- 不编造、不推测
- 回答简洁专业

=== 对话上下文（最近 10 轮）===
用户: 最近学习压力特别大
助手: 理解你的感受，这是常见的学业压力...
用户: 还是很烦该怎么办
助手: 可以尝试以下方法...
用户: 现在问题是睡不着
（仅保留最近 10 轮）

=== 知识库相关内容（精排 Top5）===
知识块 1（相关性 0.92）:焦虑症的缓解方法...
知识块 2（相关性 0.88）:学业压力的心理调适...
知识块 3（相关性 0.85）:失眠症状及建议...
知识块 4（相关性 0.82）:何时需要寻求专业帮助...
知识块 5（相关性 0.79）:...

=== 当前用户问题 ===
我最近特别焦虑，总是睡不着，该怎么办？

请严格基于上述知识库内容生成回答。
```

### 实际效果

3 个优化应用后，MindBridge 的指标显著改善：

| 指标 | 优化前 | 优化后 | 改进 |
|------|-------|-------|------|
| LLM 幻觉率 | 30% | 5% | -25% |
| 模型依赖知识库比例 | 65% | 92% | +27% |
| 用户对回答的满意度 | 68% | 82% | +14% |
| 平均 Prompt Token 消耗 | 1850 | 980 | -47% |

## 关键启示

**Prompt 工程是 RAG 系统中最容易被忽视、却最容易出问题的环节。**

我走过的弯路：
- ❌ 一开始认为 "知识库信息越多越好" → 实际上信息多反而降质
- ❌ 保留所有历史对话 → 导致 Prompt 爆炸，推理效率下降
- ❌ Prompt 约束不够明确 → 模型开始自由发挥，出现幻觉

最终的领悟：
- ✅ **Prompt 是用来约束模型的道具，而不是堆砌信息的地方**
- ✅ **少而精的信息往往比多而杂的信息效果更好**
- ✅ **明确的规则比模糊的期待更能指导模型行为**

这个经验不仅适用于 RAG，对所有大模型应用的开发都是通用的。**在 AI 时代，Prompt 工程的能力会越来越重要。**
